This website has been generated by a CMS purely written for this purpose, and is currently work in progress.

Aims for this project, according to priority are:

  1. Have a way to access the notes remotely, even over extremely slow and/or unstable networks, irrespective of the device used to access them.
  2. Explore UX and find a way to build up 'knowledge contexts' by linking different documents across webpages into a single page, and then be able to share it.
  3. Build up a way to explore contexts quickly and intuitively using a passive organizer system, paired with an exploration system.
  4. Offer a way for javascript disabled devices to access webpages effectively (not yet implemented).

Software principles to be followed are:

  • The webpage must stay as lean as possible.
  • Plug-ins installation for the explorer must be as simple as including a note in the context.
  • The software should never extent 2000loc(including comments).
    (Inspired by the style of suckless.org)
  • The architecture must stay simple enough to let it scale over itself.
  • Notes should be served in a format that is:
    • universally recognized
    • non-proprietary
    • umambiguous
    Example:html, txt
    Note: markdown is an ambiguous format
