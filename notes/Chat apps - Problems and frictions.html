Setting up group calls should be easy

Why are group calls still acting like phone calls?

I should simply click a button to create a call instance Then I quickly swipe in people into the room The invited recieve notifications in their open rooms tab. THey can at any time join the conversation as long as it's going on. Like people talking in a room. There shouldn't be a need to go only within the particular time when the ring is ringing. I'm simply entering the room. There is no time constraint on it.

Also, there should be nothing like an admin of a call. If a person leaves the room, the conversation doesn't die out. People still talk. One by one, people live and the conversation naturally dies down.

What telegram has done in their reent update is amazinglink to announcement. The group is a constantly ongoing conversation. They simply interwove text and voice, and the result seems amazing. It's something so simple, yet so beautiful. People can simply text or talk in it. Text stays frozen in time, while voice is ephemeral. That's perfect.

COming back to voice and video calls. Chat apps mostly have voice and video as second class citizens. There is no simplicity and seamlessness in that.

The user should be able to seamlessly slip in and out of chats. Like discord

  • One idea that I loved was the presenter's video superimposed on the presentation's slides. That was a dammn seamless experience. Why can't that too be the norm? What's stopping it's adoption? Lack of simple tools? or something else?
  • Video still has problems with it's choppyness. But even bad video is forgiven is the audio is crisp and clear. Have been seeing this problem on almost every remotely held podcast. There is already audio delay. But now on top of that, there is also this crappy audio quality.
    • Regarding delay: Conversations on landlines were so much better: there was barely any noticeable delay, and hence barely any talking over one another. Now there is so much delay on ip calls that the two poeple talk to each other as if they are talking on walkie talkies.
    • If there's any problem that will greatly improve the quality of communication, it has to be this delay problem.

Discord too has it's problems. It came to replace older software like teamspeak. But it has theremdous overheads: * Every thing needs a seperate server * The admin has to set up dedicated channels for dedicated things * I still see too many people simply hanging out in general * There are too many individual permissions to control and tune in order to get a community going * The channels of inetractions should be emergent, not forced * Take an example of the solution mentioned above. What did zoom get right?

What's the need of a server

  • Server is like a loud-speaker or a voice amplifier
  • p2p is fine for one on one communication, but servers solve some important problems:
    • They hold the message till the reciever gets it, even when the sender goes offline while the reciever hasn't yet come online.
      • This makes sure that the chat stays asynchronous and does not act like a phone call where everyone has to be present for communication to take place.

In group chats, servers become crucial

  • Sender can have bandwidth limits. The server acts like a loudspeaker so that all the recievers can hear the sender. A single sender's voice has a limit of loudness.

Problems I have with servers:

  • If a server is a simple holder and amplifier, why can't I host my own server.
  • I should simpmly be able to rent a server for like 5 dollars, and tell the server to watch this group
  • Once the server is approved, it simply acts as a dumb utencil
  • Beyond that, the server should be a dumb thing.

Sole cases to think about:

  • If a server watches over a group, what if some person outside the group gets invitation to join a chat room?
    • Are the servers for ephemeral chat rooms different? Do they follow the person who rented them? What happens?
    • FOllowing the renter feels stupid. It makes him a defacto admin.

Concepts I have a problem with:

  • The very concept of an admin in a group is stupid. People join a group to associate with the other poeple. Who is an admin to allow or disallow a person to come in or go out? This isn't natural
  • Defined groups by themselves are a gooey concept. What determines if a person joins a group or not? A majority vote?
    • What about the case where in agroup of poeple, people are secretly friends with the unaprooved outsider. Do they form another group chat? How does this work in real life?

I have more questions than answers.
